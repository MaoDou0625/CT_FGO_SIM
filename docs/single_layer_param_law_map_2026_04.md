## Parameter Definitions

### `road_profile_ds_m`

Distance-domain node spacing of the road-profile state.

Interpretation:

- smaller `ds` means denser spatial nodes
- denser nodes give more spatial bandwidth
- more bandwidth makes it easier to reproduce short-wave content
- too much bandwidth can enlarge IRI

This is the main parameter controlling IRI scale.

### `road_profile_anchor_sigma_m`

Low-frequency anchoring strength of the road-profile state.

Interpretation:

- smaller anchor sigma means stronger anchoring
- stronger anchoring limits low-frequency drift
- weaker anchoring gives the road-profile state more freedom

In the current experiments, this parameter is secondary.

### `gnss_sigma_vertical_m`

Vertical GNSS weighting used by the CT solver.

Interpretation:

- smaller sigma means stronger vertical GNSS trust
- larger sigma means weaker vertical GNSS trust

This parameter does not have a universal monotonic optimum. Its best value depends on the roughness level and on `ds`.

## Main Findings

### 1. `road_profile_ds_m` is the dominant scale-control parameter

This is the clearest pattern in the current map.

For both roughness levels:

- smaller `ds` tends to enlarge IRI
- larger `ds` tends to shrink IRI

Average `ct_scale` over the current grid:

For `IRI ~= 2`:

- `ds = 0.25`: `avg_scale = 1.601`
- `ds = 0.35`: `avg_scale = 1.159`
- `ds = 0.50`: `avg_scale = 0.807`
- `ds = 0.75`: `avg_scale = 0.532`

For `IRI ~= 10`:

- `ds = 0.25`: `avg_scale = 0.645`
- `ds = 0.35`: `avg_scale = 0.505`
- `ds = 0.50`: `avg_scale = 0.365`
- `ds = 0.75`: `avg_scale = 0.244`

Practical interpretation:

- `ds` acts like the main spatial-bandwidth knob
- if CT overestimates IRI, increase `ds`
- if CT underestimates IRI, decrease `ds`

### 2. `gnss_sigma_vertical_m` has a roughness-dependent optimum

The current experiments show that vertical GNSS weighting is not monotonic.

For `IRI ~= 2`, average `ct_scale` by GNSS sigma:

- `0.0080`: `avg_scale = 0.680`
- `0.0030`: `avg_scale = 0.746`
- `0.0020`: `avg_scale = 1.126`
- `0.0015`: `avg_scale = 1.547`

Interpretation for `IRI ~= 2`:

- very weak vertical GNSS tends to shrink IRI
- moderately strong vertical GNSS can give the best match
- too strong vertical GNSS drives over-amplification

For `IRI ~= 10`, average `ct_scale` by GNSS sigma:

- `0.0015`: `avg_scale = 0.330`
- `0.0010`: `avg_scale = 0.508`
- `0.0008`: `avg_scale = 0.630`
- `0.0006`: `avg_scale = 0.292`

Interpretation for `IRI ~= 10`:

- `0.0008` is currently the best region in the tested grid
- making GNSS weaker than that underestimates IRI
- making GNSS much stronger than that also degrades the result

This means the useful rule is not:

- "smaller vertical GNSS sigma is always better"

The useful rule is:

- choose GNSS sigma together with `ds`
- expect a roughness-dependent optimum

### 3. `road_profile_anchor_sigma_m` is a secondary parameter

In the current single-layer map:

- anchor changes are much weaker than `ds`
- anchor mostly provides small low-frequency stabilization
- anchor does not define the basic scaling trend

Average `ct_scale_abs_err` by anchor:

For `IRI ~= 2`:

- `anchor = 0.005`: `0.480`
- `anchor = 0.020`: `0.532`

For `IRI ~= 10`:

- `anchor = 0.005`: `0.559`
- `anchor = 0.020`: `0.561`

Interpretation:

- for `IRI ~= 10`, anchor choice is almost negligible in the current tested range
- for `IRI ~= 2`, anchor can help fine-tune some combinations, but it is still not the main driver

## Practical Rule Of Thumb

For the current simulation chain, a useful practical summary is:

1. Start by tuning `road_profile_ds_m`.
2. Use `ds` to decide whether the result should move toward more bandwidth or more smoothing.
3. After `ds` is in the correct range, tune `gnss_sigma_vertical_m`.
4. Use `road_profile_anchor_sigma_m` only as a small stabilizing adjustment.

Equivalent shorthand:

- `ds` decides the direction of IRI scaling
- `gnss_sigma_vertical_m` decides where inside that direction the final solution lands
- `anchor` only trims the result

## Current Recommendation

For continued thesis experiments, prioritize the single-layer model and study:

- finer scans of `ds`
- finer scans of `gnss_sigma_vertical_m`
- empirical relation between best `(ds, gnss_sigma_vertical_m)` and true IRI level

Do not continue the dual-layer branch as the main line unless a new observation model or new evidence clearly changes the current conclusion.
